## 📖 О проекте

**IDEF3 Editor** — это WPF-приложение для построения и анализа диаграмм в нотации **IDEF3** (методология описания процессов, *Process Description Capture*). Программа позволяет визуально собирать модель процесса из блоков-работ и перекрёстков, соединять их разными типами связей, проверять модель на структурные ошибки и раскрывать сложные блоки во вложенные диаграммы (декомпозиция).

Приложение собирается в **единственный самодостаточный `.exe`** (через Costura.Fody) — не требует установки, зависимостей и .config-файлов на Windows.

## 🚀 Быстрый старт

```bash
# 1. Клонировать репозиторий
git clone https://github.com/<your-account>/IDEF3Editor.git
cd IDEF3Editor

# 2. Открыть решение в Visual Studio
start IDEF3Editor.slnx

# 3. Собрать и запустить (F5)
```

Или собрать из командной строки:

```bash
dotnet build IDEF3Editor/IDEF3Editor.csproj -c Release
```

Готовый `IDEF3Editor.exe` появится в `IDEF3Editor/bin/Release/net48/` и запустится двойным кликом на любой машине с Windows.

## 🎛️ Панель инструментов

| Кнопка | Действие |
|---|---|
| `UOB` | Добавить блок-работу |
| `J(&)` · `J(&)s` | Перекрёсток AND (асинхр. / синхр.) |

| `Сетка` · `Привязка` | Показ сетки и привязка к ней |
| `↖ Начало` · `← Назад` | Навигация по уровням декомпозиции |

## 🧭 Работа с декомпозицией

Любой UOB-блок можно **раскрыть** в отдельную вложенную диаграмму:

1. Двойной клик по блоку → открывается его декомпозиция.
2. Стройте детализацию процесса на новом уровне.
3. Кнопкой **← Назад** возвращайтесь к родителю, **↖ Начало** — к корню.

Все уровни хранятся в едином дереве и сохраняются вместе с файлом `.idef3`.

## ✅ Проверка корректности

Встроенный валидатор (`Idef3Validator`) анализирует модель и выдаёт список замечаний трёх уровней:

- 🔴 **Ошибка** — нарушение правил нотации (например, некорректные связи перекрёстков);
- 🟡 **Предупреждение** — потенциальная проблема (например, отсутствие связей);
- 🔵 **Замечание** — информационное сообщение (например, пустая диаграмма).

Проверяются типы связей между блоками и перекрёстками, связность графа работ, минимальное число входов/выходов перекрёстков и другие структурные условия.

## 🛠️ Технологии

- **C# / WPF** — интерфейс и отрисовка холста
- **.NET Framework 4.8** (x86)
- **Newtonsoft.Json** — сериализация модели в `.idef3`
- **Costura.Fody** — упаковка зависимостей в один `.exe`

### Архитектура (ключевые файлы)

| Файл | Назначение |
|---|---|
| `MainWindow.xaml(.cs)` | Главное окно, холст, обработка мыши/клавиатуры |
| `Node.cs` | Модель и визуал блока (UOB / перекрёсток) |
| `Link.cs` | Модель и отрисовка связей (3 типа) |
| `DiagramLevel.cs` | Уровень диаграммы для декомпозиции |
| `DiagramMediator.cs` | Координация элементов диаграммы |
| `Idef3Validator.cs` | Правила проверки корректности модели |
| `UndoRedo.cs` | Менеджер отмены/повтора |
| `SaveModels.cs` | Формат сохранения `.idef3` (JSON) |
| `GridHelper.cs` | Сетка и привязка |

## 📄 Формат файла

Диаграммы сохраняются в текстовом формате **`.idef3`** — это JSON, содержащий блоки, связи, метаданные и всё дерево декомпозиций. Файлы легко читаются, сравниваются в git и переносятся между машинами.
